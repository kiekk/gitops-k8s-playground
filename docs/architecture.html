🚢 Environment Deployment Strategy

환경CI 동작ArgoCD배포 승인
Dev Image Build → ECR Push → manifest 이미지 태그 자동 수정 & Git Commit Auto Sync 없음 (완전 자동)
Prod Image Build → ECR Push → 이미지 태그 변경 PR 자동 생성 Manual Sync PR 리뷰 & 승인

🛡️ Operational Features

기능도구동작
이미지 롤백 ArgoCD Git revert 또는 argocd app rollback으로 이전 이미지 버전 복원
Auto Scale-out/in HPA CPU/Memory 기반 Pod 자동 증감 (부하 ↑ → Scale-out, 부하 ↓ → Scale-in)
임계치 알림 Prometheus + Alertmanager CPU 80%+, Memory 90%+, CrashLoopBackOff 시 Slack 알림
자동 재시작 K8s Native OOMKill (메모리 초과) 또는 Liveness Probe 실패 시 컨테이너 자동 재시작
모니터링 Grafana CPU/Memory 사용률, HPA 스케일링 이력, 배포 상태 시각화
핵심 원리: ArgoCD는 ECR(이미지 레지스트리)을 감시하지 않습니다. Git 매니페스트의 이미지 태그 변경만 감지합니다. CI 파이프라인이 이미지 빌드 후 Git 매니페스트의 이미지 태그를 업데이트해야 ArgoCD가 배포를 시작합니다.
